# ExoIntel Reproducibility Report

Reproducibility is a cornerstone of the scientific method, particularly in computational astrophysics and machine learning. This report details the explicit sequence of operations required to fully rebuild the ExoIntel platform's results from scratch.

## 1. System Requirements
The platform has been validated on Linux, macOS, and Windows environments. 
- **Python:** 3.9 or higher
- **Node.js:** 18 or higher
- **Database:** PostgreSQL 14 or higher

## 2. Environment Initialization

### 2.1 Repository Cloning
First, clone the scientific repository and navigate to the project root:
```bash
git clone https://github.com/saiiexd/exo-intel-platform.git
cd exo-intel-platform
```

### 2.2 Python Virtual Environment
Initialize an isolated environment to prevent dependency conflicts and install the rigorous machine learning and backend stack:
```bash
# Unix/macOS
python3 -m venv venv
source venv/bin/activate

# Windows
python -m venv venv
venv\Scripts\activate

# Install strictly versioned packages
pip install -r requirements.txt
```

### 2.3 Database Configuration
The pipeline demands a dedicated PostgreSQL database. You must provide a valid `DATABASE_URL` environment variable.
1. Create a logical database (e.g., `exoplanet_dw`).
2. Copy the example configuration template:
   ```bash
   cp .env.example .env
   ```
3. Update `.env` to reflect your local connection string:
   `DATABASE_URL=postgresql://user:password@localhost:5432/exoplanet_dw`

## 3. Autonomous Reproducibility Command

The ExoIntel platform is heavily orchestrated. Rather than running components individually, the central pipeline guarantees that data flows accurately from ingestion to insight generation. 

To rebuild the entire system from scratch—effectively executing the exact computational sequence utilized by the origin authors—run the following master command:

```bash
python run_exointel_pipeline.py --refresh-data --run-all
```

### What This Command Executes:
1. **NASA Data Ingestion:** Safely contacts the NASA Exoplanet Archive TAP service, downloading the latest known exoplanetary parameters and overwriting the raw database tables.
2. **Dataset Analysis & Feature Engineering:** Imputes missing values, rejects outliers, calculates the Earth Similarity Score (ESS), and creates the final enriched ML dataset.
3. **Machine Learning Model Training:** Performs a GridSearchCV over Gradient Boosting and Random Forest algorithms, locking in the preprocessor and best-performing model into a serialized `.pkl` artifact.
4. **Planet Discovery Engine:** Applies the freshly trained model against the dataset to rank the thousands of candidates, updating the `discovery_rankings` database table.
5. **Explainability Engine (SHAP):** Deconstructs the model utilizing `TreeExplainer` algorithms to output feature importance arrays and generate high-resolution analytical plots.
6. **Insight & Analytics Compilation:** Renders scientific visualizations and consolidates final reports outlining the state of the findings.

## 4. Launching the Visual Analytics Platforms

Once the autonomous pipeline concludes (verifying the backend analytics generation), the interactive discovery environments can be launched to browse the findings.

### 4.1 Streamlit Research Dashboard
For rapid, Python-native analytical reviewing:
```bash
streamlit run src/frontend/app.py
```

### 4.2 React Interactive Web Interface
For external, responsive presentation and exploration:
```bash
cd frontend
npm install
npm run dev
```
The React application (hosted on `http://localhost:5173`) will seamlessly hook into the data generated by the backend pipeline via the FastAPI service (which must be simultaneously running via `python -m src.api.main`).
